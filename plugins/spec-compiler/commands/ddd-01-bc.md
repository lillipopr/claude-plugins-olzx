# /ddd-01-bc - 限界上下文设计命令

此命令调用 **ddd-01-bounded-context-skill** Skill，从 PRD 生成限界上下文设计文档。

## 使用场景

- 已有 PRD 文档，需要生成限界上下文设计
- 需要进行业务能力分析
- 需要划分限界上下文
- 需要设计上下文映射关系

## 工作流程

```
1. 收集参数
   1.1 PRD 文档路径
   1.2 输出目录（默认为 ./output）
2. 加载 Skill：spec-compiler:ddd-01-bounded-context-skill
3. 执行工作流
   3.1 阶段 1：PRD 分析与大纲生成
        3.1.1 读取PRD
        3.1.2 提取关键信息
        3.1.3 生成大纲文件
            3.1.3.1 ddd-01-bc-outline.md
        3.1.4 提示用户
   3.2 阶段 2：讨论与优化（循环等待）
        3.2.1 等待用户编辑
        3.2.2 大纲验证
   3.3 阶段 3：详细文档生成
        3.3.1 生成详细文档
            3.3.1.1 ddd-01-bc.md
        3.3.2 最终输出
```

## 输出文件

| 文件 | 说明 |
|-----|------|
| `ddd-01-bc-outline.md` | 大纲版（可编辑讨论） |
| `ddd-01-bounded-context.md` | 详细版（定稿后生成） |

## 核心特性

- **两阶段工作流**：先生成大纲快速评审，定稿后生成详细文档
- **可编辑讨论**：大纲文件可编辑，支持迭代优化
- **质量可控**：通过人工评审确保上下文划分、领域类型的准确性

## 技术说明

**Command 类型**：直接调用 Skill（非 Agent）

**Skill 路径**：`{CLAUDE_PLUGIN_ROOT}/skills/ddd-01-bounded-context/SKILL.md`

**优势**：
- 更轻量：无需启动独立 Agent 进程
- 更快速：直接在当前会话中执行
- 更可控：Skill 包含完整流程指引，便于调试和优化
